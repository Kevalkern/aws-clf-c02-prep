# 4-Week Study Plan

This plan assumes `4 weeks` of preparation with `60-90 minutes` on weekdays and `2-3 hours` on weekends.

## Week 1: Foundations

### Day 1

- Read the official exam guide.
- Review exam domains and weightings.
- Learn Regions, Availability Zones, edge locations, and the AWS global footprint.

### Day 2

- Study cloud value propositions:
  - agility
  - elasticity
  - economies of scale
  - fault tolerance
  - global reach
- Learn CAPEX vs OPEX.

### Day 3

- Study the AWS shared responsibility model.
- Learn the six Well-Architected pillars at a high level.

### Day 4

- Learn core compute:
  - EC2
  - Auto Scaling
  - Elastic Load Balancing
  - Lambda
  - Lightsail

### Day 5

- Learn core storage:
  - S3
  - EBS
  - EFS
  - FSx
  - S3 Glacier

### Day 6

- Learn core database:
  - RDS
  - Aurora
  - DynamoDB
  - ElastiCache
  - DocumentDB

### Day 7

- Do a 25-40 question mixed quiz.
- Review every wrong answer and tag the gap:
  - concept
  - service recognition
  - wording trap

## Week 2: Security And Networking

### Day 8

- Study IAM users, groups, roles, and policies.
- Learn least privilege and MFA.
- Know when to use IAM roles instead of access keys.

### Day 9

- Study AWS Organizations, IAM Identity Center, and Control Tower.
- Understand multi-account governance at a high level.

### Day 10

- Study KMS, Secrets Manager, ACM, Shield, and WAF.
- Learn encryption at rest vs in transit.

### Day 11

- Study logging and monitoring:
  - CloudWatch
  - CloudTrail
  - AWS Config
  - Security Hub
  - GuardDuty

### Day 12

- Study VPC basics:
  - public vs private subnet
  - internet gateway
  - security groups
  - NACLs
  - Route 53

### Day 13

- Study hybrid and connectivity:
  - VPN
  - Direct Connect
  - CloudFront
  - API Gateway
  - PrivateLink

### Day 14

- Do a 50-question practice test.
- Build a mistake list and restudy weak areas.

## Week 3: Billing, Support, And Use Cases

### Day 15

- Study pricing models:
  - On-Demand
  - Reserved Instances
  - Savings Plans
  - Spot

### Day 16

- Study cost tools:
  - AWS Budgets
  - Cost Explorer
  - Cost and Usage Reports
  - Trusted Advisor

### Day 17

- Study support plans and when to use each one.
- Know differences between basic, developer, business, and enterprise-style support features.

### Day 18

- Study migration and transfer services:
  - DMS
  - Migration Hub
  - Snow Family
  - Application Migration Service

### Day 19

- Study application integration and serverless:
  - SQS
  - SNS
  - EventBridge
  - Step Functions
  - Fargate

### Day 20

- Study analytics and ML recognition:
  - Athena
  - Redshift
  - QuickSight
  - Glue
  - Kinesis
  - SageMaker AI
  - Rekognition
  - Textract

### Day 21

- Do a second 50-question practice test.
- Update your weak-topic list.

## Week 4: Exam Readiness

### Day 22

- Revisit the four domains in weighting order:
  - Cloud Technology and Services
  - Security and Compliance
  - Cloud Concepts
  - Billing, Pricing, and Support

### Day 23

- Drill service comparisons:
  - CloudWatch vs CloudTrail vs Config
  - S3 vs EBS vs EFS
  - RDS vs DynamoDB
  - IAM vs IAM Identity Center
  - SNS vs SQS
  - Shield vs WAF

### Day 24

- Review support plans, pricing, billing, and consolidated billing.
- Review Organizations and account strategy.

### Day 25

- Take a full mock exam under timed conditions.

### Day 26

- Review every missed question.
- Write one-sentence corrections for each mistake.

### Day 27

- Final light review:
  - flashcards
  - weak services
  - shared responsibility
  - pricing traps

### Day 28

- Stop heavy studying.
- Do a short confidence review only.
- Sleep early before the exam.

## Final 48-Hour Focus

- Do not cram obscure services.
- Focus on high-frequency services and decision patterns.
- Review wording around `fully managed`, `serverless`, `high availability`, `cost-effective`, and `least operational overhead`.
